# 不懂代码也能试用 CCFTS

> 给建筑业财务和管理人员看的说明。不需要任何编程知识。

---

## 1. CCFTS 到底是什么

可以这样理解：

> 它不是一个 App，更像是一套给 AI 看的工作说明书。

你平时遇到一个业务问题——比如某个项目快报资料要上报了、某项增值税跨区域预缴不知道怎么判断、某个分包结算要检查合规性——你会怎么做？查制度、翻合同、问同事、找总部。

CCFTS 就是把这些常见业务场景中"施工企业财务该怎么想、怎么看、先检查什么、后检查什么、哪些地方容易出错"，写成了一套 AI 能读懂的说明文件。你把脱敏资料和说明文件一起给 AI，AI 就能按你指定的场景来辅助分析。

---

## 2. 我应该把它装到哪里

根据不同 AI 工具，用法稍有不同。

**如果你用豆包（桌面版或网页版）**：
直接打开 `agent-packs/doubao/` 文件夹，选一个场景包，把整个文件夹拖到豆包的对话框里。再把你的脱敏资料也上传。不需要安装任何东西。

**如果你用马维斯或腾讯元宝**：
打开 `agent-packs/marvis/` 文件夹，按里面的说明操作。或者把 `docs/quick-start-10min.md` 里的说明复制粘贴到对话框。

**如果你用 Trae Solo**：
把整个下载下来的仓库文件夹拖到 Trae Solo 里。然后告诉 AI："请先读 START_HERE.md，再帮我处理我提供的脱敏资料。"

**如果你用 Claude Desktop**：
最简单的方式是把需要的技能文件和脱敏资料直接拖到对话框。如果想长期用，可以按 `mcp/SMOKE_TEST.md` 配置 MCP 接入。

**如果你用 Codex、Claude Code 或 Cursor**：
直接打开整个仓库目录，AI 可以读取所有文件。按 START_HERE.md 的场景说明选择要加载的技能。

**如果你只用手机**：
可以复制单个场景说明（从 `START_HERE.md` 选一个场景）到对话框，再上传少量脱敏资料。只适合简单场景试用，不适合上传大量资料。

## 3. 为什么我还要告诉 AI 我的工作背景

同样叫"项目"，在施工企业里可能是：

- 一个项目部（非法人，只管施工，权益接近于零，不单独纳税）；
- 一个项目公司（独立法人，有注册资本，有银行贷款，要独立纳税）；
- 一个分公司（非法人，但可能有多个项目部，内部往来复杂）；
- 一个集团合并层面（上面这些都要汇总，内部往来要相互抵消）。

AI 不知道你到底是哪一种。你不说清楚，AI 就会按最通用的口径来答，可能跟你的实际情况完全不同。

所以强烈建议：**第一次用之前，先生成一份背景包**。推荐用 [背景包生成器](../tools/context-builder/index.html)（双击用浏览器打开，在表单里填写你的信息，点「生成背景包」→「一键复制」给 AI，无需联网）。如果不用浏览器，也可以花 5 分钟手动填写 `docs/user-context-template.md`。熟悉后，可以改用更完整的 `docs/context-pack-template.md`。

## 4. 第一次怎么试（约 10 分钟）

1. **下载项目**。在 GitHub 或 Gitee 上点"下载 ZIP"，解压。
2. **打开 examples 文件夹**。随便选一个演示数据看看格式。
3. **生成背景包**。打开 `tools/context-builder/index.html`（双击在浏览器中打开），填表后「一键复制」。或者复制 `docs/user-context-template.md` 手动填写。
4. **挑一个你最熟悉的业务场景**。建议从"月度快报资料检查"开始，因为这是最普遍的需求。
5. **准备脱敏资料**。把你真实的科目余额表、合同台账等去掉单位名称和敏感金额，换成"某项目""某业主""xxx 万元"。
6. **打开 AI 工具，上传资料**。把脱敏资料和场景包一起给 AI。
7. **看 AI 输出**。检查 AI 哪些判断符合你的实际经验、哪些不符合、哪些它说"不确定"。
8. **记录结果**。用 `docs/validation-log-template.md` 记录这次试用的发现。

## 5. 推荐提示词

你可以把下面这段话复制给 AI，作为每次使用时的开头：

```text
你现在使用 CCFTS 的施工企业财税技能包。

请先阅读我提供的使用者背景资料，再阅读场景说明。

处理资料时：
1. 先判断资料是否完整，列出我缺少哪些必要资料。
2. 再按场景说明中的检查步骤逐项分析。
3. 不确定的地方必须写"不确定"，不能编造数据。
4. 不要直接给最终税务或审计结论。

输出时请分为：
- 资料缺口
- 口径判断
- 风险点
- 建议动作
- 必须人工复核的事项
```

## 6. 安全提醒

- **先用脱敏资料试**。不要第一次就把真实项目合同、真实金额、真实单位名称上传。
- **不要上传身份证号、银行账号、纳税识别号**。
- **不要让 AI 直接替你做正式税务判断**。AI 只能整理判断路径和资料缺口，最终结论必须由税务人员做出。
- **重要结论必须由人复核**。AI 说"建议这样做"不等于"必须这样做"，要结合你的实际情况和本单位制度。
- **不要上传内部保密制度全文**，除非你已获得授权。

## 7. 常见问题速查

更多问题见 [docs/faq.md](faq.md)。

- 我不会代码，可以用吗？→ 可以，这个项目就是给不懂代码的人准备的。
- 这个是软件吗？→ 不是，是一套给 AI 看的工作说明书。
- 要安装吗？→ 不需要。下载后直接用 AI 工具打开就行。
- 能上传真实项目资料吗？→ 建议先用脱敏资料。确需使用时，必须去掉单位名称、敏感金额和隐私信息。
- 输出结果能直接用于报税、审计吗？→ 不能。必须人工复核。
- 为什么同样是科目余额表，不同主体口径不同？→ 因为项目部、项目公司、分公司的法律身份和管理要求不同。详见主体判断场景。
