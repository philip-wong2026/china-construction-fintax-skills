# 如何反馈试用结果

CCFTS 现在是公开试验阶段。项目是否有价值，不能只看技能文件写了多少，更要看它在脱敏业务场景里是否能稳定帮助人发现资料缺口、口径问题和人工复核点。

如果你试用了 CCFTS，欢迎反馈。但请先确保所有内容已经脱敏。

---

## 可以反馈什么

建议反馈以下几类问题：

1. **脱敏案例验证记录**
   - 你用了哪个场景；
   - 提供了哪些脱敏资料；
   - AI 输出了什么；
   - 哪些判断正确；
   - 哪些判断不准确；
   - 哪些地方需要补规则。

2. **规则错误**
   - 某个 skill 的判断路径不对；
   - 某个税务、报表、项目管理口径容易误导；
   - 某个场景没有提醒人工复核。

3. **新手看不懂**
   - README、START_HERE、场景包、demo 哪些地方不清楚；
   - 哪些术语需要改成中文或加解释；
   - 哪一步不知道怎么操作。

4. **希望新增场景**
   - 例如资金计划、刚性兑付说明、竣工结算、分包结算、清收清欠、质保金、两金压降等。

## 不要反馈什么

请不要提交以下内容：

- 真实单位名称；
- 真实项目名称；
- 税号、身份证号、银行账号；
- 未脱敏合同原文；
- 未授权公开的内部制度全文；
- 真实敏感金额明细；
- 可以反推出具体项目或单位的附件。

## 推荐反馈方式

### 方式一：填写验证记录模板

复制 [validation-log-template.md](validation-log-template.md)，填好后再反馈。

这适合记录一次完整试用：

- 用了哪个 AI 工具；
- 用了哪个 CCFTS 场景；
- 输入了什么脱敏资料；
- AI 哪些判断对；
- 哪些判断错；
- 哪些规则需要补。

### 方式二：通过 GitHub Issue

如果你会用 GitHub，可以在项目页面选择 Issue 模板：

- 提交脱敏案例；
- 报告 skill 错误；
- 请求新增场景；
- 提交专家复核意见。

### 方式三：通过 Gitee Issue

国内用户也可以在 Gitee 仓库里提交 Issue。内容可以直接复制验证记录模板，不需要写成技术问题。

### 方式四：直接联系维护者

如果你不会使用 GitHub 或 Gitee，可以把问题整理成文字发给维护者。建议按以下格式：

```text
我试用的场景：
我的岗位：
单位层级：
项目类型：
使用的 AI 工具：
输入资料是否脱敏：
AI 输出中有用的地方：
AI 输出中不准确的地方：
希望 CCFTS 补充的规则：
```

## 反馈前自查

提交前请确认：

- 已经脱敏；
- 没有真实单位、项目、人名、税号、银行账号；
- 没有未经授权的合同或制度原文；
- 已说明使用场景；
- 已说明人工复核结果；
- 不把 AI 输出当作正式财税、审计或合规意见。
