# 常见问题

这页写给第一次看到 CCFTS 的普通用户，尤其是建筑业财务、税务、商务、项目管理人员和领导。

## CCFTS 到底是什么

它是一套“给 AI 看的施工企业财税工作说明书”。

你把它交给 AI 后，AI 在处理施工企业财务快报、增值税跨区域预缴、成本分析、清收清欠、质保金、竣工结算等问题时，会先参考这些规则和检查清单，而不是完全靠通用知识临场发挥。

## 它是一个软件吗

不是。

它不是财务软件，不是报税软件，也不是一个新的聊天机器人。它更像一套专业资料包，可以放进豆包、马维斯、Codex、Claude、Cursor、Trae Solo 等 AI 工具里使用。

## 它装到哪里

看你用什么工具：

| 你用的工具 | 最简单做法 |
| --- | --- |
| 豆包 | 上传 `agent-packs/doubao/` 里的场景包 |
| 马维斯 | 上传 `agent-packs/marvis/` 场景包，或让它读取本地文件夹 |
| Trae Solo | 打开整个项目目录 |
| Codex / Claude Code / Cursor | 直接读取仓库，或配置 MCP |
| Claude Desktop | 直接上传文件，熟悉后再配置 MCP |
| 企业知识库 | 由信息化或数字化人员按知识库资料接入 |

新手不要一开始就研究 MCP。先用场景包试一次。

## 豆包能不能用

能用，但建议用简化方式。

不要让豆包一次读取全部 96 个 skill。建议上传一个场景包，例如：

- `agent-packs/doubao/flash-report.md`
- `agent-packs/doubao/vat-prepayment.md`

然后再上传脱敏后的业务资料。

## 马维斯能不能用

理论上可以，但取决于你使用的版本是否支持读取本地文件夹、上传文件、长文本上下文等能力。

建议先用 `agent-packs/marvis/desktop-finance-review.md` 做小样本试用，不要一开始处理真实敏感资料。

## 公司电脑能不能用

可以试用，但要先遵守单位数据安全要求。

如果公司电脑只允许使用本地 AI 或单位批准的 AI 平台，就不要把真实数据上传到个人版网页 AI。可以先用脱敏样例测试流程，确认价值后再考虑内部部署。

## 能不能上传真实数据

不建议把真实敏感数据上传到公开或个人版 AI 工具。

至少要先脱敏：

- 企业名称改掉。
- 项目名称改掉。
- 税号、合同编号、银行账号、人名删掉。
- 金额可以按比例缩放。
- 不要上传未经授权公开的内部制度、合同、报表和审计资料。

## skill 和背景包有什么区别

skill 解决“AI 是否懂施工企业财税方法”的问题。

背景包解决“AI 是否懂你这一次具体工作”的问题。

例如 CCFTS 可以告诉 AI 增值税跨区域预缴通常要看哪些事实，但它不知道你这次项目在哪里、合同怎么签、有没有分包、是否简易计税。所以你还要填写 [context-pack-template.md](context-pack-template.md)。

## skill 和知识库有什么区别

普通知识库通常是把资料交给 AI 搜索。

skill 更强调“怎么做事”：适用范围、输入资料、处理步骤、输出格式、风险点、人工复核清单。

简单说，知识库更像资料柜，skill 更像作业指导书。

## 它和财务软件有什么区别

财务软件负责记账、出表、流程审批、凭证、报税接口等正式生产工作。

CCFTS 不替代这些系统。它主要辅助 AI 做资料理解、检查清单、风险提示、分析草稿和人工复核准备。

## AI 看了 skill 后，为什么还会错

因为 AI 仍然可能误读资料、忽略上下文、引用过期规则，或者把不确定的事项说得太肯定。

所以 CCFTS 强制强调：

- 缺失资料要列出来。
- 推断结论要标明。
- 风险点要提示。
- 最后必须人工复核。

## 这个项目现在成熟吗

还没有到成熟生产版。

当前是公开试验版，适合学习、脱敏试用、组织 AI 工作流、收集反馈，不适合直接用于正式纳税申报、审计签字、监管报送或重大经营决策。

## 为什么需要真实脱敏案例

没有真实脱敏案例，就只能说明“规则写得像那么回事”，不能证明它在实际业务中稳定有用。

后续最需要的是：

- 真实脱敏资料。
- AI 输出结果。
- 人工复核意见。
- 错误修正记录。
- 可重复验证的指标。

## 我不会 GitHub，怎么反馈

可以把问题整理成文字，发给项目维护者。能使用 GitHub 的话，建议直接提 issue，并选择对应模板：

- 提交脱敏案例。
- 报告 skill 错误。
- 请求新增场景。
- 提交专家复核意见。

## 最推荐的第一次试用方式

1. 打开 [quick-start-10min.md](quick-start-10min.md)。
2. 选择一个场景包。
3. 填写 [context-pack-template.md](context-pack-template.md)。
4. 上传脱敏资料。
5. 要求 AI 输出缺失资料、风险点和人工复核清单。
6. 用 [manual-review-template.md](manual-review-template.md) 复核。
