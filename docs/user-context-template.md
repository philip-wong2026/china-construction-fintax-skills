# 使用者背景资料模板

> 使用 CCFTS 前，建议先把本文件复制一份，按你的实际情况填写。它的作用是告诉 AI：你是谁、你所在单位是什么口径、这次要处理什么资料、哪些结论必须人工复核。

如果你已经熟悉 CCFTS，也可以使用更完整的 [context-pack-template.md](context-pack-template.md)。本文件是给第一次试用的新手准备的简化版。

## 8. AI 输出要求

请 AI 按以下格式输出：

1. 资料缺口
2. 口径判断
3. 风险点
4. 建议动作
5. 必须人工复核的事项

额外要求：

- 不确定的地方必须写“不确定”。
- 资料不足时先列资料缺口，不要编造。
- 不直接给最终税务、审计或合规结论。
- 正式报告正文不要出现文件路径、内部变量名、source_path、调试信息。

## 10. 给 AI 的启动提示词

```text
请先阅读我的使用者背景资料，再阅读 CCFTS 场景说明。

在正式分析前，请先用简短文字确认：
1. 你理解的使用者岗位；
2. 本次主体是项目部、项目公司、分公司、子公司还是集团层面；
3. 本次任务属于哪个场景；
4. 当前资料是否足够；
5. 哪些结论必须人工复核。

未经确认，不要直接输出最终结论。
```
