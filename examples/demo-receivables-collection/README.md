# Demo：往来款分层清收管理样例

## 解决什么管理问题

这个 demo 不是教会计做账，也不是让 AI 判断坏账。它模拟某建设集团拿到一张脱敏应收账款台账后，如何让 AI 先做管理分层：

- 哪些项目应优先催收；
- 哪些项目先补资料、先确权，再谈催收；
- 哪些项目存在争议，需要商务、项目部、法务或清收部门介入；
- 哪些事项必须由人工复核，不能由 AI 直接下结论。

施工企业清收清欠的难点，通常不是“有没有欠款”，而是不同项目的欠款状态完全不同。有的已确权但回款慢，有的未确权导致不能直接催，有的开票和计量不匹配，有的争议尚未解决。只按金额大小排序，容易把管理资源用错地方。

## 输入文件

| 文件 | 说明 |
|------|------|
| `input-desensitized.md` | 脱敏应收账款台账，所有项目、金额、部门均为虚构 |
| `expected-output.md` | 期望 AI 输出的管理分析结构 |
| `validation-log.md` | 人工复核记录模板 |

## AI 应该先检查什么

1. 账龄区间：区分 `0-90天`、`90-180天`、`180天以上`。
2. 当前状态：区分已确权、未确权、争议中。
3. 计量、开票、回款是否匹配。
4. 责任部门是否明确。
5. 哪些项目应先催收，哪些项目应先补资料或解决争议。
6. 哪些事项必须人工复核。

## 输出边界

AI 可以输出分层催收优先级、资料缺口、风险标记和建议动作。

AI 不得直接输出坏账判断、核销结论、诉讼结论、审计结论或正式清收责任认定。

## 适合谁使用

- 清收清欠岗；
- 项目财务；
- 商务合约人员；
- 项目经理部管理人员；
- 总部经营分析或财务管理人员。

## 当前限制

- 本 demo 全部数据均为虚构整数，不代表任何真实项目。
- 本 demo 只用于说明往来款分层清收的管理分析路径。
- 实际项目还需要结合合同条款、业主资信、结算争议、地方关系、后续合作和法务意见。
