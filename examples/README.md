# 脱敏 Demo

本目录提供三个可复现的脱敏示例。它们不是企业真实报表，也不代表正式财税意见。

这三个 demo 的价值不是“检查报表本身有多深”，而是先让新用户看懂 CCFTS 的基本工作方式：

1. AI 不能拿到资料就直接下结论，必须先识别主体、期间、资料范围和复核边界。
2. 项目部、项目公司、税务事项的责任主体不同，不能用同一套判断逻辑。
3. AI 输出必须列出缺失资料、风险点和人工复核清单。

第一批 demo 先选“项目部”“项目公司”“增值税跨区域预缴”，是因为它们分别代表三类最容易混淆的基础场景：

- **项目部/总包部**：不是独立法人，很多问题重在项目管理和内部考核。
- **项目公司**：通常是为项目设立的独立法人公司，涉及资产、负债、权益和税务义务。
- **增值税跨区域预缴**：牵涉项目所在地、机构所在地、计税方式和分包扣除，不能只看一个税率。

后续应继续补充更有管理提升价值的 demo，例如清收清欠、资金计划、竣工结算、质保金到期回收、项目经营分析和亏损项目预警。

## Demo 1：项目部 / 总包部

目录：`demo-project-unit-flash-report/`

适合：

- 非法人项目部
- 总包部
- 施工总承包类主体
- 项目财务或报表人员

为什么值得看：

- 看 AI 如何先判断“这是项目部/总包部，不是独立法人公司”
- 看 AI 如何避免把项目部内部管理资料当成公司法人口径处理
- 看 AI 如何输出资料缺口、异常点和人工复核清单

## Demo 2：项目公司

目录：`demo-spv-flash-report/`

适合：

- 独立法人项目公司
- 为项目设立的投资管理类公司
- 子公司或项目公司财务

为什么值得看：

- 看 AI 如何先判断“这是独立法人项目公司”
- 看 AI 如何区分项目公司和项目部的责任主体、税务义务、权益关系
- 看 AI 为什么不能把项目公司问题简单当成项目部问题处理

## Demo 3：增值税跨区域预缴

目录：`demo-vat-prepayment/`

适合：

- 税务岗
- 项目财务
- 涉及跨区域施工项目的共享中心人员

为什么值得看：

- 如何区分一般计税和简易计税
- 如何先追问项目所在地、机构所在地、合同类型、计税方式和分包扣除凭证
- 如何提示项目所在地附加税费和地方口径复核
- 如何避免“只看税率、不看业务事实”的错误

## 推荐使用方式

1. 先读 [docs/quick-start-10min.md](../docs/quick-start-10min.md)
2. 填一份 [项目背景包](../docs/context-pack-template.md)
3. 选择一个 demo 目录
4. 把 demo 的 `README.md`、`input/` 和相关 skill 交给 AI
5. 对照 `expected/` 检查 AI 输出是否接近

## 注意

Demo 的价值不是证明 AI 一定正确，而是提供一个可练习、可复核、可讨论的最小样本。正式业务使用前，必须换成你的脱敏资料，并由具备专业能力的人员复核。
