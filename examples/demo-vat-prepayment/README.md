# Demo: 增值税跨区域预缴检查

## 目的

演示 AI 在判断施工项目是否需要增值税跨区域预缴时，不能只看税率，而要先核实项目所在地、机构所在地、计税方式、合同类型、分包扣除凭证和地方税务口径。

## 场景

某施工企业有一个外省公路工程项目，需要判断是否涉及增值税跨区域预缴。项目可能同时涉及一般计税、简易计税、分包款扣除和附加税费。

这个场景的管理价值在于：很多税务风险不是算错一个数字，而是前提事实没问清。例如项目所在地和机构所在地是否一致、合同是否支持简易计税、分包扣除凭证是否合规、地方税务机关是否有特殊要求。

## 输入文件

| 文件 | 路径 | 说明 |
|------|------|------|
| 项目合同信息 | `input/project-info.csv` | 脱敏教学样例：项目地址/合同金额/计税方式/分包信息 |
| 期望预缴计算 | `expected/prepayment-calculation.csv` | 基于样例信息手工整理的 expected 输出 |

## 应加载的技能（按顺序）

1. `ccfts-intel-sta-vat-law-2026`
2. `ccfts-tax-all-vat-cross-region-prepayment`
3. `ccfts-tax-all-vat-general-filing`
4. `ccfts-tax-all-vat-simplified-filing`（如含简易计税部分）

## 期望输出

| 输出项 | 内容 |
|--------|------|
| 计税方式判定 | 一般计税部分：预征率 2%；简易计税部分：预征率 3% |
| 预缴税额计算 | 一般：(价款−分包款)/(1+9%)×2%；简易：(价款−分包款)/(1+3%)×3% |
| 附加税费 | 城建税 + 教育费附加 + 地方教育附加（按项目所在地税率） |
| 预缴台账 | 按项目核算的完整预缴记录 |

## 如何判断结果合理

- 一般计税预征率 2%，简易计税 3%——不可互换
- 分包款扣除凭证必须是增值税发票（非白条/收据）
- 附加税费按项目所在地城建税税率（7%/5%/1%），非机构所在地
- 预缴税额 ≤ 机构所在地申报的应纳税额（否则有退税 or 留抵）

## 当前限制

- 本 demo 全部数据均为脱敏教学样例，只用于演示流程。
- 本 demo 覆盖预缴税额计算和附加税费跟随计算，不覆盖金税四期三方比对和电子税务局实际操作。
- 各省税务机关对跨区域预缴的具体操作流程可能有差异，本 demo 基于通用规则。
