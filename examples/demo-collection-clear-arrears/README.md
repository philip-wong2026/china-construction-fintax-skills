# Demo: 清收清欠台账分析与催收优先级

## 目的

演示 AI 在处理清收清欠台账时，不能只看欠款金额大小，而要综合判断：账龄长短、是否确权、是否已开票、是否已回款、责任部门是否明确、催收状态、资料是否齐全。输出催收优先级清单、风险分层、建议动作和人工复核事项。

## 场景

某施工企业有 10 个处于不同清收阶段的脱敏项目：有的账龄超过 3 年但尚未完成竣工结算；有的已发律师函但业主经营困难；有的属于政府项目但财政拨款滞后；有的处于正常回款期。

这个场景的管理价值在于：清收清欠的难点往往不是"有没有人催"，而是"该不该催、催谁、催不回来怎么办"。只看应收账款余额大小而忽略账龄分层、业主类型、确权状态和资料完备性，会导致资源错配——对"该走法律程序的"还在发催收函，对"就差最后一步的"反而没人跟进。

## 输入文件

| 文件 | 路径 | 说明 |
|------|------|------|
| 应收/合同资产/回款台账 | `input/collection-ledger.csv` | 脱敏教学样例：10 个项目的应收台账，包含账龄、确权、开票、回款、催收状态、责任部门、资料完备性 |
| 期望催收优先级 | `expected/collection-priority.csv` | 基于台账手工整理的 expected 输出：优先级排序、风险分层、建议动作、升级路径 |

4. `ccfts-workflow-base`：通用工作流

## AI 应该检查什么

| 检查维度 | 说明 |
|---------|------|
| 欠款账龄 | 按 <6 个月、6 个月-1 年、1-2 年、2-3 年、>3 年分层。账龄越长，回收难度越大，手段应该越硬 |
| 是否已确权 | 竣工结算是否完成、审计是否审定。未确权的债权在法律上无法强制执行 |
| 是否已开票 | 已开票是正常催收的前提（部分业主需要发票才能安排付款） |
| 是否已回款 | 区分"累计收款"和"累计结算"的差额，已到期未付部分才是催收重点 |
| 责任部门 | 财务部、商务部、清欠办、法务部——每个阶段的牵头部门不同，缺失责任人说明组织保障不明确 |
| 催收状态 | 电话沟通→催收函→现场催收→高层对接→律师函→诉讼→财产保全——手段是否与账龄匹配 |
| 资料缺口 | 缺验收单、缺审计审定单、缺竣工决算批复——这些缺口会导致无法确权或无法进入下一阶段 |
| 高风险项目 | 民企业主（信用风险高）、>3 年未确权（时效风险）、质保金占比高（回收周期长） |

## 期望输出

| 输出项 | 内容 |
|--------|------|
| 催收优先级清单 | 按紧急程度排序（先解决"能催但没人催的"，再解决"催了但手段不够的"，最后处理"催不了需要先补资料的"） |
| 风险分层 | 黑色（账龄长、未确权、资料缺口严重且回收路径不清）、红色（2-3 年或民企业主信用风险较高）、橙色（1-2 年，或城投/政府类确权、拨款、质保金事项需专项跟踪）、黄色（6 个月-1 年常规逾期）、绿色（<6 个月正常回款期） |
| 建议动作 | 针对每个项目的下一步具体行动：继续等、发函、现场催收、高层对接、律师函、诉讼、暂停催收先补资料 |
| 责任部门 | 明确牵头部门和配合部门 |
| 升级路径 | 如果当前手段无效，下一步做什么、触发条件是什么 |
| 人工复核事项 | 哪些结论必须由人确认后才能行动 |

## 如何判断结果合理

- 账龄 > 2 年且已穷尽常规催收手段的项目，不应排在"发催收函"——手段应该与账龄匹配
- 未确权的项目，不应直接建议"诉讼"——应优先推动确权（竣工结算/审计审定）
- 民企业主的项目，风险分层通常应高于同等账龄的政府项目（信用风险差异）
- 质保金的回收节奏应单独评估，不应与工程款混合排序
- 已发律师函但未回复的项目，应升级而非重复发函

## 当前限制

- 本 demo 全部数据均为脱敏教学样例，只用于演示流程和判断路径。
- 本 demo 假设各项目的合同信息、业主资信与台账一致，实际情况中可能需要额外调查。
- 催收优先级仅基于台账信息排序，实际排序还应考虑：业主关系、后续项目合作、地方政策等因素。
- 本 demo 不覆盖以资抵债、债务重组、核销的具体操作流程。
- 不构成法律建议。法律手段（诉讼、保全、强制执行）必须咨询专业律师。

## 参考

- 国资委/央企应收账款管理关注点：应收账款周转率、账龄结构、坏账计提充足性
- 政府项目催收的特殊性：不能轻易起诉，需通过上级主管部门、财政部门、审计途径推动
- 诉讼时效风险：长期未回款项目需要由法务或律师确认权利基础、起算点、中断证据和具体适用期限
