这是一套给 AI 使用的施工企业财税工作说明书。它的作用是：帮助 AI 在处理快报、税务、成本、合同、清欠、正式报告时，**先按施工企业的口径检查，再交给人工复核**。

它不是软件，不是报税工具，不是替代会计师或税务师的东西。

## 这个项目能帮什么忙

| 场景 | 适合谁 | 怎么用 |
|------|--------|--------|
| 月度快报资料检查 | 项目财务、总部财务 | 上传脱敏科目余额表，让 AI 先检查资料完整性、口径一致性、异常指标，再上报 |
| 项目经营分析和亏损预警 | 项目经理、商务、经营分析人员 | 用脱敏成本、收入、合同数据，让 AI 按责任成本口径做偏差分析，标记亏损风险 |
| 清收清欠台账分析 | 项目财务、清欠人员 | 上传脱敏应收台账，让 AI 按账龄分层、业主类型分类，输出催收优先级清单 |
| 增值税跨区域预缴判断 | 税务岗、项目财务 | 输入项目地址、合同信息、计税方式，让 AI 整理判断路径和资料缺口 |
| 分包结算风险检查 | 项目商务、财务 | 用脱敏分包合同和结算单，检查合同范围、甲供材扣回、质保金扣留是否合规 |
| 竣工结算财税复核 | 项目商务、财务 | 上传送审金额和审定金额，让 AI 检查收入调整、合同资产转应收、质保金和红字发票 |
| 把企业内部经验整理成 AI 可读的知识包 | 数字化部门、财务共享中心 | 参照 skills 目录结构，把本企业的管理口径、检查清单写成 AI 可读的文件 |
| 生成正式报告初稿 | 需要写报告的人 | 让 AI 按规范格式输出检查报告、情况说明或汇报材料初稿，人工修改后定稿 |

## 这个项目**不能**做什么

- **不替代财务人员**：只能辅助检查，不能替代财务判断和审批。
- **不替代税务意见**：不构成正式税务申报意见。税务结论必须由税务人员和税务师确认。
- **不替代审计结论**：不能替代年报审计、专项审计或内部审计。
- **不直接处理未脱敏敏感数据**：不要上传真实单位名称、金额、合同原件、身份证号、银行账号。
- **不保证所有企业口径一致**：每个企业的管理口径可能不同，AI 按通用行业规则判断，需要人工确认是否符合本单位制度。
- **必须结合本企业制度和人工复核**：AI 的输出只是"参考资料"，最终决定必须由人做出。

## 谁适合使用

- **施工企业财务人员**：项目财务、总部财务、报表岗、共享中心核算岗。
- **项目商务和成本人员**：项目经理部商务人员、成本合约人员。
- **经营分析人员**：公司总部经营分析岗、财务分析岗。
- **数字化部门**：负责企业 AI 应用、知识管理、流程优化的人员。
- **想让自己公司的 AI 更懂建筑业管理口径的人**：通过阅读这些技能文件的结构和方法，你可以把本企业的经验也整理成类似的知识包。

## 不懂代码怎么试

这不是程序，不需要安装。基本用法只需要四个步骤：

**第一步：下载项目。**
在 GitHub 或 Gitee 上找到这个仓库，点"下载 ZIP"，解压到你的电脑上。

**第二步：找到适合你的入口。**
- 如果你用豆包、马维斯、腾讯元宝：打开 `agent-packs/` 文件夹，选一个场景包上传。
- 如果你每次只需要处理一个具体业务：打开 `docs/quick-start-10min.md`，照着做就行。
- 如果你想让 AI 更了解你的工作背景：用 [背景包生成器](tools/context-builder/index.html)（无需联网，在浏览器里填表就行），或者复制 `docs/user-context-template.md` 手动填写。

**第三步：把脱敏资料和场景包一起给 AI。**
打开你常用的 AI 工具，上传场景包文件夹或复制场景说明，再上传你的脱敏资料。

**第四步：检查 AI 的输出。**
AI 输出后对照人工判断，用 `docs/manual-review-template.md` 逐项检查，记录差异。

**重要提醒**：第一次试用时，不要上传真实项目资料。先用 `examples/` 里的脱敏演示数据试试，确认流程没问题再用自己的脱敏资料。

## 支持哪些 AI 工具

| 工具 | 适合怎么用 | 限制 |
|------|-----------|------|
| 豆包 Desktop / Web | 上传场景包文件夹，上传脱敏文件，复制提示词 | 适合单个场景试用，不适合一次性加载全部 97 个技能文件 |
| 腾讯元宝 / 马维斯 | 上传文件夹或粘贴场景说明和脱敏资料 | 同上，注意单次上传文件大小限制 |
| Trae Solo | 打开整个仓库文件夹，按场景说明让 AI 读取对应文件 | 适合本地使用，不依赖网络 |
| Claude Desktop | 上传文件到对话，或配置 MCP 接入 | MCP 接入需要额外配置，适合长期使用 |
| Claude Code / Codex / Cursor | 直接打开仓库目录，AI 可以读取所有文件 | 需要一定的 AI 编程工具使用经验 |
| 手机端 AI | 复制单个场景说明和脱敏资料到对话框 | 只适合简单场景试用，不适合复杂资料 |

## 第一次试用建议（约 10 分钟）

1. 打开 `examples/` 文件夹，随便选一个演示数据看看 AI 能做什么。
2. 复制 `docs/user-context-template.md`，填 5-10 行你的基本背景（岗位、单位类型、项目类型、本企业管理口径）。
3. 挑一个你最熟悉的业务场景（建议从"月度快报资料检查"开始）。
4. 用脱敏资料让 AI 处理一个小场景。
5. 对照你自己的判断，检查 AI 哪些说对了、哪些不对、哪些遗漏。
6. 用 `docs/manual-review-template.md` 或 `docs/validation-log-template.md` 记录结果。
7. 如果觉得有用，再逐步扩展到更多场景。

## 项目里有什么

| 目录 | 是什么 | 谁需要看 |
|------|--------|---------|
| `skills/` | AI 可读的技能文件（97 个），覆盖快报、税务、成本、合同、清欠、合并、分析、管理 | 想让 AI 处理具体业务的人，或者想了解行业通用口径的人 |

| `mcp/` | 高级接入方式（MCP 服务），适合长期在 Claude Desktop 等工具中使用 | 熟悉 AI 工具配置的高级用户 |
| `tests/` | 项目自检脚本，确保技能文件结构正确 | 开发者 |
| `scripts/` | 批量检查工具 | 开发者 |

## 项目状态

**当前版本定位：v0.4.0-alpha 公开试验版。**

**当前阶段：公开试验阶段（Public Alpha）。**

这意味着：
- 技能文件的结构和内容已经过初步验证，可供试用。
- 但它还没有经过大范围的实际业务检验。
- 也没有经过注册会计师或税务师的正式审核。

**当前状态：**
- **技能文件**：97 个，覆盖快报、税务、成本、合同、清欠、合并、分析、管理等场景。
- **脱敏演示场景**：5 个，覆盖项目部快报、项目公司口径、增值税跨区域预缴、清收清欠台账分析、往来款分层清收。
- **面向新手的文档**：`docs/nontechnical-user-guide.md`、`docs/user-context-template.md`、`docs/prompt-for-ai-agent.md`、`docs/validation-log-template.md`、`docs/feedback.md`。
- **工具接入说明**：已补充豆包、马维斯、腾讯元宝、Trae Solo 等常见 AI 工具的使用入口。

**目前最缺**：
- **更多脱敏案例**：现在已有五个演示场景，但仍缺少来自不同企业、不同项目类型、不同管理口径的脱敏案例。
- **更多人工复核记录**：AI 输出和人工判断的对比记录，是提高可信度的关键。
- **不同企业口径的反馈**：你的企业可能和写这些技能文件时假设的口径不同。请告诉我们哪里不一样。
- **更真实的场景验证**：用脱敏后的真实项目资料跑一遍，验证流程是否顺畅。

如果你愿意贡献脱敏案例或反馈，请先看 [docs/feedback.md](docs/feedback.md)，再填写 [docs/validation-log-template.md](docs/validation-log-template.md)。
