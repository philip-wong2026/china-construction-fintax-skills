# 豆包场景包：施工企业快报检查

## 这个文件怎么用

1. 打开豆包桌面端或网页端。
2. 上传本文件。
3. 上传一份已经脱敏的科目余额表。
4. 复制下面的提示词发给豆包。
5. 用输出最后的“人工复核清单”逐项检查。

## 适用场景

- 项目部月度快报
- 项目公司月度资料
- 子公司单户快报初稿检查
- 科目余额表到快报项目的映射检查

## 你需要提供

- 科目余额表
- 报表期间
- 主体说明：项目公司、项目部、子公司或集团
- 是否季度末、年末
- 如有企业原报表，请提供脱敏对照结果

## 豆包需要遵守的 CCFTS 规则

- 先判断主体类型，再做报表映射。
- 不确定主体类型时，先列出缺失信息，不要直接下结论。
- 输出必须区分：确定事实、推断结论、风险点、人工复核项。
- 资产负债表必须检查是否平衡。
- 不得把 AI 输出包装成最终财务报送意见。

## 复制给豆包的提示词

```text
你现在使用 CCFTS（中国施工企业财税技能包）的快报检查规则，帮我处理一份脱敏科目余额表。

请严格按以下步骤输出：
1. 先判断主体类型：项目公司 / 项目部 / 子公司 / 集团 / 无法判断。
2. 说明判断依据和置信度。
3. 列出缺失信息。
4. 给出科目到快报项目的映射思路。
5. 输出利润表草稿和资产负债表草稿，单位万元。
6. 检查资产负债表是否平衡。
7. 列出异常科目、可能的期末调整和风险点。
8. 最后输出“人工复核清单”。

重要限制：
- 不要把不确定事项当成确定结论。
- 不要把输出作为最终合规意见。
- 如果数据不足，请先要求我补充信息。
```

## 人工复核重点

- 主体类型是否与真实组织架构一致
- 科目余额方向是否正确
- 是否存在未结转收入、成本、税金
- 其他应付款、合同资产、合同负债口径是否合理
- 资产负债表是否平衡
- 是否存在敏感数据未脱敏
