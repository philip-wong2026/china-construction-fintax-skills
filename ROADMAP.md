# 路线图

CCFTS 当前是公开试验版。路线图不再用版本号暗示完成进度，而是用状态标签标明事项成熟度：

- `[已完成]`：已经进入公开仓库，并经过基础验证。
- `[进行中]`：已有初步内容，但还需要继续补强、验证或收集反馈。
- `[计划中]`：方向明确，但尚未形成可稳定试用的内容。

版本号只作为发布记录使用，不代表路线图顺序。

## 已完成

### [已完成] 公开试用基础

对应发布：`v0.1.0-public-alpha` 起，后续版本持续修订。

已完成内容：

- 97 个施工企业财税 skill。
- README、START_HERE、新手说明、10 分钟上手、FAQ。
- 豆包、马维斯、腾讯元宝、Trae Solo 等常见 AI 工具接入说明。
- `agent-packs/` 场景包，便于普通用户直接上传给 AI。
- 本地 MCP server 和基础 smoke test。
- `scripts/validate-skills.py`、demo 合同检查、MCP 加载检查等基础验证脚本。
- GitHub Actions 基础校验。

### [已完成] 普通用户试用入口

对应发布：主要在 `v0.4.0-alpha` 完成公开化补强。

已完成内容：

- 面向不懂代码用户的说明：`docs/nontechnical-user-guide.md`。
- 使用者背景模板：`docs/user-context-template.md`。
- 可复制给 AI 的启动提示词：`docs/prompt-for-ai-agent.md`。
- 试用记录模板：`docs/validation-log-template.md`。
- 反馈与贡献入口：`docs/feedback.md`。
- README 改为中文主导，说明“这是什么、能做什么、不能做什么、怎么试”。

### [已完成] 脱敏教学 demo

已完成内容：

- 项目部快报资料检查 demo。
- 项目公司经营口径识别 demo。
- 增值税跨区域预缴判断 demo。
- 清收清欠台账分析与催收优先级 demo。
- 往来款分层清收管理样例 demo。

当前定位：

- 这些 demo 主要用于说明 CCFTS 的工作方式。
- 目前仍以脱敏教学样例为主，不能证明在所有企业口径下均可直接适用。

## 进行中

### [进行中] 清收清欠场景做深

下一步目标：

- 将现有清收清欠 demo 从“教学样例”推进到“更接近实务验证样例”。
- 增加更多脱敏台账样式，例如政府/城投/国企/民企业主、质保金、长期未确权、已发律师函、诉讼前准备等。
- 记录 AI 输出、人工复核结果、错因和修正规则。
- 沉淀“催收优先级、风险分层、升级路径、人工复核事项”的判断标准。

### [进行中] skill 语义验证

下一步目标：

- 不只验证 skill 文件结构，还要验证关键业务结论是否合理。
- 为高风险、高频 skill 建立小型问答对或案例测试集。
- 用固定场景检查 AI 是否能输出关键追问、资料缺口、风险提示和人工复核边界。
- 优先覆盖：月度快报、增值税跨区域预缴、清收清欠、项目公司/项目部主体判断、竣工结算财税复核。

### [进行中] 多工具使用反馈

下一步目标：

- 用同一份脱敏资料，在豆包、马维斯、Trae Solo、Claude、Codex 等工具中对比输出。
- 记录不同工具在文件读取、上下文理解、表格处理、中文表达、复核提醒方面的差异。
- 明确哪些工具适合普通用户试用，哪些工具适合长期接入。

## 计划中

### [计划中] 专家复核机制

目标：

- 邀请 CPA、税务师、审计人员、施工企业财务/商务/清欠专家，对核心场景进行人工复核。
- 将案例区分为“教学样例”“实务脱敏样例”“专家复核样例”。
- 对争议事项标明适用边界，不把不确定口径写成确定结论。

### [计划中] 管理场景继续补强

优先方向：

- 资金计划。
- 竣工结算。
- 质保金到期回收。
- 项目经营分析。
- 亏损项目预警。
- 两金压降。
- 项目现金流和垫资风险。

原则：

- 不再单纯增加 skill 数量。
- 每补一个场景，应同时补脱敏输入、期望输出、人工复核点和局限说明。

### [计划中] 企业内部知识库和 agent 接入

目标：

- 整理企业知识库接入建议。
- 补充本地 MCP 和可能的 HTTP/SSE 接入说明。
- 形成内部权限、脱敏、日志、复核、责任边界建议。
- 补充企业试点流程：小范围脱敏试用、专家复核、再扩大范围。

## 稳定版目标

达到稳定版前，希望具备：

- 主要 skill 有明确来源、适用范围和人工复核点。
- 关键场景有脱敏案例和复核记录。
- 高频场景有语义验证样例，而不只是文件结构验证。
- 普通用户能按文档完成一次低风险试用。
- 专业用户能按技能索引找到所需文件。
- 企业用户能判断是否适合进入内部试点。
- 项目边界、许可证、免责声明、贡献规则清楚。

## 当前最需要外部帮助的地方

- 提供可公开的脱敏业务样例。
- 指出施工企业实务口径中的错误。
- 反馈豆包、马维斯、Trae Solo、Codex、Claude 等工具的真实使用体验。
- 提供专家复核意见。
- 补充更有管理深度的场景。
