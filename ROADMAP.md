# 路线图

CCFTS 当前是公开试验版。路线图的重点不是堆更多文件，而是让项目逐步变得更可信、更好用、更容易被普通建筑业从业者试用。

## v0.1：公开试用版

状态：已发布。

已完成：

- 96 个施工企业财税 skill。
- README、新手说明、10 分钟上手、背景包模板。
- 豆包、马维斯、Trae Solo 场景包。
- 3 个脱敏教学 demo。
- 本地 MCP server。
- 基础结构验证脚本和 GitHub Actions。

当前不足：

- 脱敏 demo 仍偏教学样例，不足以证明管理提升效果。
- 专家复核记录不足。
- 普通用户贡献入口刚开始建立。
- 部分工具接入方式仍需要实际用户反馈。

## v0.2：补强普通用户试用

目标：让不懂 GitHub、不懂代码的人也能完成一次低风险试用。

计划：

- 完善豆包、马维斯、Trae Solo 的图文式使用说明。
- 增加普通用户 FAQ。
- 补充更多场景包，例如清收清欠、质保金到期回收、资金计划、亏损项目预警。
- 增加“如何准备脱敏资料”的示例。
- 建立 issue 模板，让用户能按固定格式反馈。

## v0.3：补强验证和实验

目标：从“看起来有用”推进到“有可复核证据”。

计划：

- 建立脱敏案例记录。
- 建立验证指标，例如资料补全率、风险识别率、人工复核命中率、输出可用性。
- 对每个 demo 记录输入、输出、人工复核、问题修正。
- 邀请 CPA、税务师、审计人员或建筑企业财税专家参与复核。
- 区分“教学样例”“实务脱敏样例”“专家复核样例”。

## v0.4：补强管理场景

目标：从财税检查扩展到更有管理价值的项目经营场景。

计划优先补充：

- 清收清欠。
- 资金计划。
- 竣工结算。
- 质保金到期回收。
- 项目经营分析。
- 亏损项目预警。
- 两金压降。
- 项目现金流和垫资风险。

## v0.5：企业内部知识库和 agent 接入

目标：让企业信息化、数字化、财务共享中心可以把 CCFTS 接入内部 AI 系统。

计划：

- 整理企业知识库接入建议。
- 补充本地 MCP 和可能的 HTTP/SSE 接入说明。
- 形成内部权限、脱敏、日志、复核、责任边界建议。
- 补充企业试点流程：小范围脱敏试用、专家复核、再扩大范围。

## v1.0：稳定版目标

达到 v1.0 前，希望具备：

- 主要 skill 有明确来源、适用范围和人工复核点。
- 关键场景有脱敏案例和复核记录。
- 普通用户能按文档完成一次试用。
- 专业用户能按技能索引找到所需文件。
- 企业用户能判断是否适合进入内部试点。
- 项目边界、许可证、免责声明、贡献规则清楚。

## 当前最需要外部帮助的地方

- 提供可公开的脱敏业务样例。
- 指出施工企业实务口径中的错误。
- 反馈豆包、马维斯、Trae Solo、Codex、Claude 等工具的真实使用体验。
- 提供专家复核意见。
- 补充更有管理深度的场景。
