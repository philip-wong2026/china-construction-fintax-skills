# 贡献指南

感谢你关注 CCFTS。本项目欢迎施工企业财务、税务、商务、项目管理、数字化、审计、咨询和 AI agent 使用者参与。

## 可以贡献什么

目前最需要这几类贡献：

- 脱敏业务案例：快报、增值税跨区域预缴、企业所得税、清收清欠、资金计划、竣工结算、质保金、亏损项目预警等。
- 使用反馈：哪个场景看不懂、跑不通、不实用。
- 规则纠错：法规、准则、税务口径、施工企业实务口径中的错漏。
- 专家复核：CPA、税务师、审计人员、建筑企业财税专家对 skill 的复核意见。
- 接入经验：豆包、马维斯、Trae Solo、Codex、Claude、Cursor、企业知识库等工具的实际使用方式。

## 不要提交什么

请不要提交：

- 未脱敏的企业名称、项目名称、税号、合同编号、银行账号、身份证号、人名、手机号、地址等敏感信息。
- 企业内部制度、模板、报表、合同、审计资料、投标资料、经营数据等未经授权公开的资料。
- 能够反推出具体企业、具体项目、具体人员的信息。
- 任何你没有权利公开的资料。

如果不确定能不能公开，请不要提交。

## 脱敏基本要求

提交案例前，请至少做到：

- 企业名称改成“某施工企业”“A 公司”等。
- 项目名称改成“X 项目”“某高速项目”等。
- 税号、合同号、银行账号、人名全部删除。
- 金额可以按比例缩放，但要保持表内勾稽关系。
- 日期可以保留月份或季度，也可以平移处理。
- 说明哪些字段是真实结构，哪些字段做过缩放或替换。

## 提交脱敏案例建议格式

建议包含：

```text
1. 场景名称：
2. 适用对象：
3. 已脱敏资料清单：
4. 希望 AI 完成什么：
5. 人工复核发现：
6. 当前 CCFTS 输出哪里有问题：
7. 是否允许项目维护者把该案例整理成 demo：
```

## 提交规则纠错建议格式

建议包含：

```text
1. 涉及文件：
2. 问题位置：
3. 你认为哪里不准确：
4. 建议改法：
5. 依据来源：
6. 是否需要区分地区、企业制度或适用期间：
```

## 提交方式

如果你不会写代码，直接在 GitHub Issues 里选择对应模板即可：

- 提交脱敏案例
- 报告 skill 错误
- 请求新增场景
- 提交专家复核意见

如果你会使用 GitHub，也可以提交 Pull Request。提交前建议运行：

```bash
python3 scripts/validate-skills.py
python3 tests/test_demo_contracts.py
python3 tests/test_skill_integrity.py
python3 tests/test_mcp_loading.py
```

## 维护者会如何处理

维护者会优先处理：

- 能提升公开可信度的脱敏案例。
- 会导致 AI 输出错误结论的规则问题。
- 普通用户高频使用场景。
- 有 CPA、税务师、审计人员或建筑企业专家复核依据的问题。

本项目仍处于公开试验版，所有内容都需要人工复核后使用。
