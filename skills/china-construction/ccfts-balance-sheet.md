# CCFTS — 资产负债表编制规则

## 快速参考

| 项目 | Type A（投资管理类） | Type B（施工总承包类） |
|---|---|---|
| 取整模式 | ROUND_HALF_UP | 全部 ROUND_DOWN |
| 合同资产(1483/1484) | 组件级 ROUND_HALF_UP 后相减 | 组件级 ROUND_DOWN 后相减 |
| 其他应付款 | 倒推：负债合计 − 税费 − 薪酬 | ROUND_DOWN(2241-01) + ROUND_DOWN(3001) |
| 合同负债 | 无 | ROUND_DOWN(5801 贷方余额) |
| 长期应付款 | 无 | ROUND_DOWN(2701) |
| 权益合计 | 正常权益结构 | ≈ 0（资产 = 负债） |
| 资产 = 负债 + 权益？ | 是 | 是（权益 ≈ 0，资产 ≈ 负债） |

## 一、资产侧映射

### Type A（投资管理类）

| 快报项目 | 公式 | 注意 |
|---|---|---|
| 货币资金 | Σ ROUND(1002 各子行) | 区分内部财务公司 vs 外部金融机构 |
| 其中：内部财务公司 | ROUND(1002 财务公司子行) | |
| 其中：外部金融机构 | Σ ROUND(1002 非财务公司子行) | |
| 预付账款 | ROUND(1123 期末借方余额) | |
| 其他应收款 | ROUND(1221) − ROUND(1231-02) | 净额 = 账面 − 坏账准备 |
| 流动资产合计 | 以上各项合计 | |
| 合同资产 | Σ ROUND(1483 子行) − Σ ROUND(1484 子行) | 组件级分别取整 |
| 其他非流动资产 | 合同资产 + ROUND(1531净) | |
| 固定资产净额 | ROUND(1601) − ROUND(1602) | 原值 − 折旧 |
| 递延所得税资产 | ROUND(1811 期末借方余额) | |
| 非流动资产合计 | 以上非流动项目合计 | |
| **资产总计** | 流动 + 非流动 | |

### Type B（施工总承包类）

| 快报项目 | 公式 |
|---|---|
| 货币资金 | Σ ROUND_DOWN(各银行账户) |
| 其他应收款 | ROUND_DOWN(1221) − ROUND_DOWN(2241-29) |
| 流动资产合计 | 货币资金 + 其他应收款 |
| 合同资产 | ROUND_DOWN(1483) − ROUND_DOWN(1484) |
| 其他非流动资产 | ROUND_DOWN(1531净) + 合同资产 |
| 非流动资产合计 | 同上 |
| **资产总计** | 流动 + 非流动 |

## 二、负债侧映射

### Type A（投资管理类）

| 快报项目 | 公式 | 注意 |
|---|---|---|
| 应付职工薪酬 | ROUND(2211 期末贷方余额) | |
| 应交税费 | ROUND(2221 期末净额) | |
| 其他应付款 | **倒推**：负债合计 − 应交税费 − 应付职工薪酬 | 不直接取 2241！ |
| 流动负债合计 | 以上各项合计 | |
| 非流动负债 | 0（或按实际情况） | |
| **负债合计** | 流动 + 非流动 | |

> **为什么 2241 要倒推？** 直接取 2241 期末余额在临界值（末五位 5000 元）时会产生 ±1 万元差异。倒推消除了这个不确定性。已验证：倒推结果与实际快报完全一致。

### Type B（施工总承包类）

| 快报项目 | 公式 |
|---|---|
| 应付账款 | ROUND_DOWN(2202) − ROUND_DOWN(1123) |
| 合同负债 | ROUND_DOWN(5801 贷方余额) |
| 其他应付款 | ROUND_DOWN(2241-01) + ROUND_DOWN(3001) |
| 其他流动负债 | ROUND_DOWN(1126) |
| 流动负债合计 | 以上各项合计 |
| 长期应付款 | ROUND_DOWN(2701) |
| 非流动负债合计 | 同上 |
| **负债合计** | 流动 + 非流动 |

## 三、权益侧映射

### Type A
| 快报项目 | 公式 |
|---|---|
| 实收资本 | ROUND(4001 期末贷方余额) |
| 资本公积 | ROUND(4002 期末贷方余额) |
| 盈余公积 | ROUND(4101 期末贷方余额) |
| 未分配利润 | ROUND(4105-14 期末贷方净额) |
| **权益合计** | 以上各项合计 |

### Type B
权益合计 ≈ 0（内部总包部结构，资产 = 负债）。

## 四、平衡验证

```
资产总计 = 负债合计 + 所有者权益合计
```

| 主体 | 验证方式 |
|---|---|
| Type A | 资产总计 ≈ 负债合计 + 实收资本 + 资本公积 + 盈余公积 + 未分配利润（允许 ±1 万元取整噪音） |
| Type B | 资产总计 ≈ 负债合计（权益 ≈ 0） |

若不满足平衡，排查顺序：
1. 1483/1484 是否做了组件级分项取整？
2. 其他应付款是否正确使用了倒推（Type A）？
3. 取整模式是否正确（Type A ROUND_HALF_UP vs Type B ROUND_DOWN）？

## 五、自检清单

1. [ ] 货币资金是否为各子行分别取整后汇总？
2. [ ] 1483/1484 是否为组件级取整？
3. [ ] Type A 2241 是否使用倒推而非直接取数？
4. [ ] Type B 资产负债表所有行是否使用 ROUND_DOWN？
5. [ ] Type B 权益是否 ≈ 0？
6. [ ] 资产 = 负债 + 权益？（允许 ±1 取整噪音）
7. [ ] 固定资产是否为 1601 − 1602？
