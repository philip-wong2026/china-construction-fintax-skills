# 全电发票——施工企业要点

## 快速参考

| 项目 | 时间/要求 |
|---|---|
| 全国强制推行 | **2026年Q2** |
| 核心特征 | 无纸化、云端实时上传、不可离线 |
| 与纸质发票关系 | 纸质发票**逐步退出**（过渡期并存） |
| 施工行业特别要求 | 备注栏必填项：**项目地址+项目全称** |

## 与传统电子发票的区别

| 项目 | 传统电子发票 | 全电发票 |
|---|---|---|
| 介质 | 版式文件(PDF/OFD) | 数据电文（云端） |
| 开具方式 | 税务局UKey/税盘 | 电子发票服务平台直开 |
| 交付方式 | 邮件/下载 | 系统自动推送至购买方数字账户 |
| 红冲流程 | 提交税务审批 | 开票方发起→购买方系统中确认（双向确认） |
| 作废 | 可作废当月发票 | **不可作废**，只能红冲 |

## 施工行业增值税发票备注栏要求

### 建筑服务发票（必填）

```
备注栏必须包含：
1. 建筑服务发生地（精确到县/市/区）
2. 项目名称（与施工许可证或合同一致）
```

**示例**：
> 项目地址：某省某市某区xxx路
> 项目名称：xx项目总承包工程

### 备注栏不规范的后果

- 金税四期自动比对 → 字段缺失则标记为**异常发票**
- 购买方取得的异常发票 → **不得抵扣进项税额**
- 反复出现 → 列为**风险纳税人**

## 分包发票管理

- 总包方向发包方全额开票（含分包款）
- 分包方向总包方开票
- 差额纳税时，分包发票作为差额扣除凭证
- 清包工/甲供工程的**分包部分普通发票限制**（2026年增值税法）

## ERP 系统对接

施工企业需将 ERP（如用友/金蝶/SAP）与全电发票平台对接：
- 自动获取购买方收到的发票数据
- 自动完成三单匹配（采购订单、入库单、发票）
- 自动计算可抵扣进项税额
