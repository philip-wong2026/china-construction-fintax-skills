# 金税四期——施工企业要点

## 快速参考

| 项目 | 状态 |
|---|---|
| 金税四期 | **2026年全面运行** |
| 数据源 | 40+ 政府部门互联互通 |
| 全电发票 | **2026年Q2全国强制推行** |
| 施工企业核心监控 | "五流合一"智能比对模型 |

## "五流合一"智能比对

金税四期对**每一笔施工交易**进行五流交叉验证：

| 流 | 数据来源 | 比对目标 |
|---|---|---|
| 合同流 | 住建部合同备案 | 签约主体 vs 开票主体 vs 收款主体 |
| 货物流 | 供应商物流/GPS | 钢材/混凝土等实物流向 |
| 发票流 | 全电发票平台 | 票面金额 vs 合同金额 vs 资金流 |
| 资金流 | 银行流水 | 付款账户 vs 开票账户 |
| **工程流** | 住建部施工许可/进度 | **施工进度 vs 发票金额 vs 材料用量**（施工专属） |

**五流中任何一流与其余四流不匹配 = 自动告警。**

## 施工行业高风险信号

| 信号 | 触发条件 |
|---|---|
| 钢材用量异常 | 发票 5000 吨钢材 / 10000 ㎡ 项目 → 可能虚开发票 |
| 合同对手与资金流不一致 | 发票抬头 A 公司，付款至 B 公司 → 可能虚开发票 |
| 发票缺失必填备注 | **建筑服务发票**必须备注：项目地址（精确到区县）+ 项目全称 |
| 人工成本做编码 | 金税四期已取消人工成本编码，误用会触发虚开发票警告 |
| 项目地与实际经营地不一致 | 异地施工未做跨区域涉税报告 |

## 全电发票（2026年Q2全国强制）

| 特征 | 说明 |
|---|---|
| 云端实时上传 | 开票即上传，不可离线 |
| 税务数字账户 | 直连企业 ERP 系统 |
| 红字确认双向验证 | 开票方发起红冲 → 受票方必须在平台确认 |
| 供应链发票关系图 | 税务系统自动构建上下游发票网络 |
| 必填备注（建筑服务） | 项目地址（精确到区县）+ 项目全称 |

## 施工企业合规要点

1. **每张开出的建筑服务发票**：备注栏必须完整填写项目所在地 + 项目全称
2. **每张收到的材料发票**：确认供应商与合同签订方一致
3. **异地施工**：先在项目地做跨区域涉税报告，再开票
4. **分包发票**：清包工/甲供工程——分包部分的差额征税凭证要备查
5. **人工费**：做好实名制登记，与个税申报联动
