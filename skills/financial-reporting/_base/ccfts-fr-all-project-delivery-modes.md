# 施工企业多项目交付模式对比

> **SLOT 说明**：本文件提供四种主流项目交付模式的全面对比。加载后根据项目实际情况选择对应模式。

## 快速参考：四种模式一表对比

| 维度 | 纯施工 | EPC | 专项债项目 | PPP/BOT |
|------|--------|-----|-----------|---------|
| 承包范围 | 仅施工 | 设计+采购+施工 | 施工（或EPC） | 投资+设计+施工+运营 |
| 施工角色 | 承包商 | 总承包商 | 承包商 | **股东+EPC承包商** |
| 收入 | 6001 | 6001+6051 | 6001 | 6001 + 6111 + 运营收入 |
| VAT | 9% | 6%+13%+9% | 9% | 9%+13%+6%（按业务） |
| 履约义务 | 单一 | 需判断 | 单一 | 多义务（建设+运营） |
| 合同资产规模 | 中 | 大（含E/P未结算） | 中 | 很大（含PPP项目资产） |
| 是否合并项目公司 | 否 | 否 | 否 | **按CAS 33判断** |
| 资产负债率影响 | 低 | 中 | 低 | **高**（如合并PPP SPV） |
| 回款风险 | 业主信用 | 业主信用 | 财政拨付延迟 | 运营期收益风险 |

## 一、纯施工模式（基准）

```
施工企业 → 施工合同 → 业主
                         │
                    按进度结算
                    收入：6001
                    VAT：9%
```

**特点**：最简单，收入确认、VAT、合同资产均为标准处理。
所有现有技能文件的默认假设。

## 二、EPC 模式

```
施工企业(总承包) → EPC合同 → 业主
    │
    ├── E(设计)：6% VAT
    ├── P(采购)：13% VAT
    └── C(施工)：9% VAT
```

### 收入确认决策

```
EPC 合同
  │
  ├── E/P/C 高度关联、不可拆分？
  │   └── YES → 单一履约义务 → 按整体进度确认
  │
  └── E/P/C 可明确区分？
      └── YES → 多项履约义务 → E/P/C 分别确认
          ├── E：交付设计成果时
          ├── P：设备到场验收时（注意！不等安装完成）
          └── C：按时段（投入法/产出法）
```

### 关键差异点

| 项目 | 纯施工 | EPC |
|------|--------|-----|
| 收入确认时点 | 仅施工进度 | E/P 可能早于 C 确认 |
| 设备部分（P） | 不涉及 | 设备到场即确认（不等安装） |
| 设计部分（E） | 不涉及 | 交付成果确认 |
| 利润分布 | 均匀 | 可能前高后低（P 利润在前期） |
| 增值税发票 | 仅 9% | 需分别开具 6%/13%/9% |

### EPC 模式下质保金的拆分

EPC 合同可能对不同组成部分设置不同的质保条款：
- E（设计）：通常无单独质保金
- P（设备）：供应商质保（2-5年）+ 总承包商对业主的质保
- C（施工）：标准 3% 质保金，1-2 年缺陷责任期

## 三、专项债项目模式

```
省级财政 → 专项债发行
  ↓
市/县财政 → 项目单位（政府/事业单位/城投）
  ↓
施工企业 ← 施工合同 ← 项目单位
```

### 与纯施工的关键差异

**相同**：
- 收入确认方式完全一致（按时段，CAS 14）
- 科目映射一致（6001/6401/5801/5601）
- 增值税处理一致（9% 建筑服务）

**不同**：
- 业主是政府单位/事业单位（非企业），结算单据格式可能不同
- 收款周期可能更长（财政审批链）
- 坏账风险：信用风险低但**流动性风险高**（延迟 ≠ 不付）
- 合同资产挂账时间可能更长

### 专项债项目的合同资产披露

由于政府客户的付款依赖于财政预算审批和债券资金拨付，
合同资产（已完工未结算）的挂账周期可能显著长于商业项目。
应在报表附注中披露"合同资产中 XX 万元涉及政府专项债项目，收款进度取决于财政拨付"。

## 四、PPP/BOT 模式

```
施工企业(社会资本)
  │
  ├── 股权投资 → PPP项目公司(SPV)
  │                  │
  │                  ├── 银行贷款
  │                  ├── 专项债/补助
  │                  └── 运营收入
  │
  └── EPC合同 → PPP项目公司(SPV)
```

### PPP 的三层会计问题

| 层次 | 问题 | 准则依据 |
|------|------|---------|
| 第 1 层：EPC 施工收入 | 何时确认、如何计量 | CAS 14（时段法） |
| 第 2 层：SPV 是否合并 | 控制三要素判断 | CAS 33 + 国资委要求 |
| 第 3 层：SPV 自身的会计 | 金融资产 vs 无形资产模式 | 解释第 14 号 |

### 第 1 层：EPC 施工收入——与普通 EPC 一致

即使 SPV 是关联方（施工企业控股），施工收入的确认也不能提前：
- 必须按时段确认（投入法/产出法）
- 不能因为"SPV 是我控股的"就提前确认全部收入
- **关联交易需在合并层面抵销**

### 第 2 层：SPV 合并判断——施工企业最关键的决策

```
施工企业是 SPV 最大股东？
  ├── YES → 国资委要求原则性合并
  │         → 合并后资产、负债、收入大幅增加
  │         → 资产负债率显著上升
  └── NO → 权益法核算
           → SPV 资产/负债表外
```

**核心矛盾**：合并 → 收入规模大 + 资产规模大，但资产负债率也高（PPP 项目负债率通常 70-80%）。

### 第 3 层：SPV 的资产分类

| 回报机制 | 资产类别 | 收入确认 |
|---------|---------|---------|
| 政府付费 | 长期应收款（金融资产） | 利息收入（按实际利率法） |
| 使用者付费 | 无形资产—特许经营权 | 运营期摊销 |
| 可行性缺口补助 | 金融资产 + 无形资产 | 分别处理 |

### PPP 未覆盖的技能空白

本文件仅提供模式对比框架。以下专项技能待开发：
- PPP 项目公司（SPV）的独立会计核算（含金融资产/无形资产模式）
- PPP 建设期利息资本化
- PPP 运营期收入确认（政府付费 vs 使用者付费 vs 混合）
- PPP 项目移交阶段的资产处置

## 五、模式选择决策矩阵

施工企业选择项目模式的考量：

| 考量因素 | 纯施工 | EPC | 专项债项目 | PPP |
|---------|--------|-----|-----------|-----|
| 利润空间 | 低（纯施工利润） | 中（E/P 有额外利润） | 中 | 高（施工+投资+运营） |
| 资金占用 | 低 | 中（设备垫资） | 低 | **高**（股权投资+前期投入） |
| 回款周期 | 短 | 中 | 中-长 | **很长**（运营期 10-30 年） |
| 出表需求 | 不适用 | 不适用 | 不适用 | **敏感**（资产负债率考量） |
| 专业能力要求 | 施工 | 设计管理+采购管理 | 政府关系 | 投融资+运营 |
