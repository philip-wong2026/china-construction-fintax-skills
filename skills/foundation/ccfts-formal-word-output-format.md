# CCFTS 正式 Word 输出格式

## 适用范围

凡通过 CCFTS 技能形成正式 Word/PDF 文本交付件，默认加载本技能。适用文档包括报告、请示、情况说明、函、通知、会议材料、经济活动分析、财务分析、预算执行、成本管控、资金计划、清收清欠、项目经营分析、制度办法和管理建议。

本技能尤其适用于中国中铁及所属单位、工程项目、项目总包部、项目公司等正式材料。

不适用于 MBA、thesis、学术论文、论文数据分析、答辩材料、合同、招投标文件、外部单位专用模板、学校或期刊模板，以及用户明确指定其他格式的文档。

## 使用顺序

1. 先完成 CCFTS 业务技能要求的数据核对、口径判断、风险提示和人工复核清单。
2. 若用户要求生成 Word/PDF 或正式文稿，将本技能作为最终排版和正文边界规则加载。
3. 若用户或通知指定了其他模板，优先执行用户或通知指定模板。
4. 若属于中国中铁及所属单位、工程项目、项目总包部、项目公司相关正式 Word 材料，优先复制用户已确认的正式模板作为底版，不从空白文档手工猜排版。

## 正文边界

正式报告正文只写可报送正文。来源文件、`source_path`、数据口径、关键假设、缺失材料、未确认事项和人工复核清单默认作为正文之外的起草校核说明或任务回复内容，不写入正式报告正文；用户明确要求写入正文或附件时除外。

正文段落之间不加 Markdown 式空段。标题后保留必要空行；“特此报告。”后、落款前按盖章需要空三行。

不主动添加联系人和联系方式，除非用户明确要求或具体文种要求必须列示。

## Word 格式硬性要求

默认使用中铁内部行文排版：

- 页边距：上 37mm、下 35mm、左 28mm、右 26mm。
- 行距：固定值 28 磅。
- 标题：方正小标宋简体，22pt/二号，居中，不加粗；标题前不得留空段，标题后空一行。
- 主送单位：仿宋_GB2312/仿宋GB2312，16pt/三号，固定 28 磅，左对齐，无首行缩进。
- 正文及正文内数字：仿宋_GB2312/仿宋GB2312，16pt/三号，固定 28 磅，两端对齐，首行缩进 32pt。
- 一级标题“一、”：黑体，16pt，不加粗，固定 28 磅，首行缩进 32pt。
- 二级标题“（一）”：楷体_GB2312/楷体，16pt，不加粗，固定 28 磅，首行缩进 32pt。
- 三级标题“1.”和四级标题“（1）”：仿宋_GB2312/仿宋GB2312，16pt，不加粗。
- 落款和日期：仿宋_GB2312/仿宋GB2312，16pt，右对齐，无首行缩进。
- 附件说明不加粗，与正文一致；附件正文中“附件”二字及附件序号用三号黑体，其余与正文一致。
- 版记放在偶数页；抄送单位使用四号仿宋体。
- 日期与文号不得出现虚位；发文字号年份使用六角括号，例如 `〔2026〕`。

## 项目群财务报告口径

处理工程项目群相关财务报告、经营分析、经济活动分析、资金分析、成本管控、预算执行、清欠回款、项目公司财务情况说明时，默认采用“先总后分”组织方式：

1. 先写项目群整体情况。
2. 再按各单项工程、项目总包部、项目公司等主体展开。
3. 再按收入、成本、资金、投资、税务、保险、预算、风险整改等专题分析。
4. 最后形成问题、措施和下一步建议。

不得把总包部、项目公司、参建单位、业主、发包人、承包人、施工单位混写；不得把待确认事项写成确定事实。

## 输出检查

交付正式 Word/PDF 前检查：

1. 是否已判断本技能适用，且不存在用户指定的其他模板。
2. 是否使用用户确认模板作为底版，或已按本技能设置页面、字体、字号、行距和层级。
3. 正文是否无 Markdown 式空段。
4. 正文是否未夹带 `source_path`、关键假设、未确认事项和人工复核清单。
5. 金额、日期、单位、主体名称和文种是否与 CCFTS 业务技能输出一致。
6. 文号年份是否使用六角括号，日期和文号是否无虚位。
7. 版记、附件、落款、盖章空行是否符合正式行文需要。

## 规则来源

本技能提炼自 AI Workspace 中的共享正式工作文档规则和项目群财务报告写作模板。若本技能与用户明确要求、外部通知模板或项目专用模板冲突，优先级为：用户明确要求 > 外部通知或上会模板 > 项目专用模板 > 本技能。
